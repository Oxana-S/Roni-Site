# Сайт Ростислава Вишневского — документация

Многостраничный сайт писателя-фантаста с лендингом под каждую книгу.
4 языка (RU / EN / UK / BG), тёмная «эпическая» тема, единые шапка и подвал.

> **Статус:** дизайн и фронтенд готовы полностью, включая состояния форм («успех»).
> Бэкенд (приём писем, подписка, оплата, выдача книги) **не подключён** — это точки интеграции, см. раздел «Формы» и «Чек-лист запуска».

---

## 1. Что это за проект

Сайт-«хаб» автора + переиспользуемый шаблон лендинга для книг. Идея: автор пишет серию книг,
у каждой — свой лендинг по одному шаблону; общие разделы (об авторе, блог, новости, контакты) живут
на сайте один раз.

Контент построен вокруг романа **«Просто вернуться домой»** (мир Толуса, род Даргонов): воин под
видом антиквара на Земле; боевые искусства, магия, философия долга.

---

## 2. Как это технически устроено

- Каждая страница — **Design Component**: один файл `Имя.dc.html`, который **открывается прямо в браузере**.
  Сборка (webpack/npm) не нужна.
- Внутри файла три части: **шаблон** (разметка), **класс логики** (`class Component extends DCLogic`) и
  необязательные настройки. Рантайм — файл `support.js` (служебный, **не трогать**).
- Стили — **инлайновые** (внутри `style="..."`). Шрифты — Google Fonts: `Forum` (заголовки) и
  `Spectral` (текст).
- Переходы между страницами — обычные ссылки `<a href="Имя.dc.html">`.

## 4. Общие шапка и подвал (как менять меню)

Меню и подвал вынесены в `SiteHeader.dc.html` и `SiteFooter.dc.html` и подключаются на каждой странице
двумя строками:

```html
<dc-import name="SiteHeader" lang="{{ lang }}" active="books" on-lang="{{ setLang }}" hint-size="100%,66px"></dc-import>
...контент страницы...
<dc-import name="SiteFooter" lang="{{ lang }}" hint-size="100%,320px"></dc-import>
```

- `active` — какой пункт меню подсветить: `books / author / blog / news / contact` (или пусто).
- **Добавить/переименовать пункт меню** → редактируется `SiteHeader.dc.html`: массив `NAV` (порядок и ссылки)
  и объект `CH` (названия на 4 языках). Меняется в одном месте — применяется на всём сайте.
- **Подвал** (соцсети, контакты, разделы) → `SiteFooter.dc.html`, объект `CH`.

## 5. Мультиязычность (RU / EN / UK / BG)

- Тексты каждой страницы лежат в её классе логики в объекте `DICT` (по ключу языка: `ru`, `en`, `uk`, `bg`).
- Выбранный язык хранится в браузере (`localStorage`, ключ `vish_lang`) и сохраняется при переходах.
- Переключатель языка — в шапке.

**Как добавить новый язык** (например, польский `pl`):
1. В `SiteHeader.dc.html` → массив `LANGS` добавить `{code:'pl',label:'PL'}`; в `CH` и в `NAV`-подписях — переводы.
2. В `SiteFooter.dc.html` → `CH` добавить блок `pl`.
3. В каждой странице → в `DICT` добавить блок `pl` со всеми ключами (проще скопировать `ru` и перевести).

## 6. Формы — как работают и как подключить бэкенд

Сейчас все формы **полностью оформлены и показывают экран «успех»**, но данные **никуда не отправляются**
(заглушка на фронтенде). Чтобы заработало по-настоящему, нужно в обработчик отправки добавить вызов вашего
бэкенда (или стороннего сервиса). Везде это метод `onSubmit` в классе логики страницы.

### 6.1. «Письмо автору» — `Контакты.dc.html`
Поля: имя, e-mail, сообщение. Обработчик:
```js
onSubmit:(e)=>{ e.preventDefault(); this.setState({sent:true}); }
```
**Подключение:** перед `setState({sent:true})` отправить данные, например:
- Простой путь без своего сервера — сервис форм (Formspree, Getform, Web3Forms): `fetch('https://…', {method:'POST', body:…})`.
- Свой бэкенд — `POST` на ваш эндпоинт, который шлёт письмо на `rostvish.bg@gmail.com`.

### 6.2. Подписка — `Блог.dc.html`
Поле: e-mail. Обработчик:
```js
onSub:(e)=>{ e.preventDefault(); this.setState({subbed:true}); }
```
**Подключение:** перед `setState` отправить e-mail в вашу рассылку (Mailchimp, Sendinblue/Brevo, MailerLite — у всех есть API/endpoint подписки).

### 6.3. Оформление заказа / оплата — `Заказ.dc.html`
Поля: e-mail, способ оплаты (карта/PayPal), данные карты. Обработчик:
```js
onSubmit:(e)=>{ e.preventDefault(); this.setState({paid:true}); }
```
**Подключение (важно — деньги):** поля карты сейчас визуальные. Реальную оплату делают через провайдера,
который сам показывает безопасную форму карты:
- **Stripe Checkout** или **PayPal** — самый простой и безопасный путь: кнопка ведёт на страницу провайдера,
  после оплаты — возврат на `Заказ.dc.html` (экран «успех»).
- После успешной оплаты бэкенд должен **отправить файл книги (PDF/EPUB) на указанный e-mail** — это и есть
  «выдача товара». Можно автоматически (вебхук провайдера → письмо с вложением/ссылкой).
- Реквизиты заказа уже собраны в `state` (`email`, `method`).

> Никогда не принимайте номер карты на своём фронтенде — только через форму провайдера (PCI-DSS).

## 7. Воронка продаж (путь пользователя)

```
Главная (знакомство, крючок)
   └─→ Лендинг книги
         hero (обложка, рейтинг, «Купить», Amazon, «Читать главу»)
         → о книге / «что внутри»
         → два мира
         → первые строки  ──→ Читать онлайн (бесплатный отрывок) ──┐
         → отзывы          ──→ Отзывы                              │
         → об авторе                                               │
         → ОФФЕР (цена, что входит) ── «Купить» ──→ Заказ.dc.html ←┘
                                                       └─→ экран «успех» (книга на почту)
```
Магнит — бесплатная первая глава; альтернативный канал — Amazon Kindle.

## 8. Картинки, видео, ассеты

- Картинки лежат в `assets/img/`. Сейчас это **плейсхолдеры** (тёмные градиенты с подписью файла).
- **Заменить** = положить реальный файл с тем же именем (jpg/webp, < 400 КБ). Сайт подхватит автоматически.
- Подробные промты для генерации каждой картинки — в файле **`Промты для картинок.md`**.
- Имена: `hero.jpg`, `world-earth.jpg`, `world-tolus.jpg`, `book-1.jpg`, `book-2.jpg`, `author.jpg`,
  `blog-1.jpg … blog-6.jpg`. Аудио главы (голос автора) — `assets/audio/chapter-1.mp3`.
- **Видео автора** (`Об авторе.dc.html`): в классе логики есть строка
  `VIDEO_ID = 'DePmzRg5h4g';` — впишите туда ID ролика с YouTube (часть после `v=` или `youtu.be/`).

## 9. Как добавлять контент (без бэкенда)

Контент пока лежит списками внутри страниц — добавить запись = скопировать блок и поменять текст.

- **Новость** (`Новости.dc.html`) → массив `entries` (в каждом языке): `{date, tag, text, img}`. Новые — сверху.
  Лента сама группирует по годам и прячет старое под «Показать ещё».
- **Статья блога** → карточка в `Блог.dc.html` (`posts`) + полный текст в `Статья.dc.html` (`ARTICLES` по `id`).
- **Новая книга** → скопировать `Книга - Просто вернуться домой.dc.html`, поменять тексты/цену/обложку;
  добавить карточку на Главную (витрина) и пункт в каталог.

> Для удобного редактирования при запуске рекомендуется подключить источник контента
> (Google-таблица / Notion / CMS) — тогда автор ведёт ленту и блог сам.

## 11. Дизайн-система и стайл-гайд

Живой стайл-гайд — страница **`Дизайн-система.dc.html`**. Помимо палитры и типографики, на ней есть разделы: **иконографика**, **шаблоны страниц** (лендинг, статья, лента, контент-страница, список с фильтром, служебная) и **шаблоны секций/блоков** (кикер+заголовок, карточка, сетка, оффер, CTA, таймлайн-запись, отзыв, «два мира») — чтобы переиспользовать, а не изобретать заново. Кратко по основам:

**Цвет.** Фон `#14110D` (база) / `#100D0A` (секции) / `#0D0B08` (подвал); карточки и медиа `#1C1812`.
Текст `#ECE4D6` → `#C9BDA8` (ссылки/вторичный) → `#9C917C` (приглушённый) → `#6B6152` (слабый).
Акцент — золото `#C9A86A` (кнопки, ссылки), метки-кикеры `#B8935A`, активное состояние `#F0D8A8`;
вторичный акцент — «красное солнце» Толуса `#D2683F`.

**Шрифты.** `Forum` — заголовки, цифры, бренд (антикварные капители). `Spectral` — текст (300/400, курсив для цитат).
Кикер-метки: Spectral 11px, letter-spacing 5px, золотом `#B8935A`.

**Кнопки.** Основная (золотой фон, текст тёмный) · вторичная (золотой контур) · Amazon-чип · текст-ссылка с подчёркиванием.

**Поля форм.** Тёмная подложка, рамка золотом, фокус — рамка `#C9A86A`. У всех форм есть экран «успех».

**Ритм.** Скругления 2px (кнопки) / 3–4px (карточки) / 50% (иконки). Поля страницы 7vw, контент ≤1180px.
Тени мягкие тёмные. Фирменный мотив — светящийся **золотой шов** (портал между мирами).

> Правило: новые элементы собираются из этих токенов и компонентов — тогда страница остаётся частью того же мира.

## 14. Адаптивность (мобильные устройства)

Подход — адаптив «из примитивов», без отдельной мобильной вёрстки:
- **Типографика** — `clamp()`: плавно масштабируется от мобильного к десктопу.
- **Блоки** — `flex-wrap` и `grid auto-fit / auto-fill`: колонки сами складываются в одну на узких экранах.
- **Поля страницы** — `7vw` (резиновые).
- **Меню** — общий `SiteHeader` на ширине **≤ 820px** сворачивается в кнопку-бургер (☰) с выпадающим списком; на десктопе — обычная строка.
- **Картинки** — `object-fit:cover` в контейнерах с `aspect-ratio`.

Всё новое собирайте из этих примитивов — адаптив получится сам. Дизайн-система, стайл-гайд и шаблоны также переключаются на все 4 языка.
