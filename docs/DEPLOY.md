# Деплой на сервер

Сайт статический (HTML + CSS + JS, без бэкенда). Подходит любой статический
хостинг/сервер: Apache, nginx, Caddy, Netlify, Vercel, и т.д.
Все внутренние пути относительные — сайт работает из корня любого домена без правок.

## Шаги

### 1. Прописать домен (одна команда)
Перед загрузкой укажите реальный домен — обновятся canonical, Open Graph,
sitemap и robots (для корректного SEO и превью в соцсетях):

```bash
python3 scripts/set-domain.py https://ваш-домен.com
```

Команду можно запускать повторно при смене домена. Текущий домен хранится в `.sitebase`.

### 2. Загрузить файлы в корень сайта
Загрузите содержимое папки проекта в web-root сервера (`public_html`, `www`, и т.п.).

Нужны для работы сайта:
- все `*.html`
- `support.js`, `styles/`, `components/`, `assets/`
- `favicon.ico`, `site.webmanifest`, `robots.txt`, `sitemap.xml`

Можно НЕ загружать (служебное, на работу не влияет): `docs/`, `scripts/`,
`*.md`, `.sitebase`, `.nojekyll` (нужен только для GitHub Pages).

### 3. Включить Google Analytics (когда будет)
Откройте `assets/analytics.js`, впишите ваш Measurement ID:
```js
var GA_ID = "G-XXXXXXXXXX";
```
Аналитика заработает сразу на всех страницах. Пока ID пустой — ничего не грузится.

> ⚠️ **GDPR/EU:** автор в Болгарии (ЕС). GA с cookie по закону требует согласия
> пользователя ДО запуска. Если нужно — сделаем баннер согласия и будем
> инициализировать GA только после «принять».

## Настройка сервера

- **HTTPS** обязательно (canonical и Open Graph используют `https`).
- **Страница 404:** настройте сервер отдавать `404.html` при ненайденных адресах.
  - Apache (`.htaccess`): `ErrorDocument 404 /404.html`
  - nginx: `error_page 404 /404.html;`
- **MIME для манифеста** (если иконка PWA не подхватывается):
  отдавать `.webmanifest` как `application/manifest+json`.
- Сайт обращается к **внешним ресурсам**: шрифты Google Fonts. Интернет у
  посетителя нужен (React/Babel уже лежат локально в `assets/vendor/`).

## После деплоя (SEO)

1. Добавьте сайт в **Google Search Console** (вашим Google-аккаунтом).
2. Отправьте sitemap: `https://ваш-домен.com/sitemap.xml`.
3. Проверьте превью ссылок: Facebook Sharing Debugger / Twitter Card Validator.

## Заметки

- **Многоязычные мета** (title/description/keywords) меняются по выбранному
  языку посетителя через `assets/seo-meta.js`. Статичные RU-мета в `<head>` —
  фолбэк для краулеров и OG-скраперов (они часто не исполняют JS).
- Для полноценного мультиязычного SEO (отдельные URL на язык + `hreflang`)
  нужна доработка — обсуждается отдельно при необходимости.
