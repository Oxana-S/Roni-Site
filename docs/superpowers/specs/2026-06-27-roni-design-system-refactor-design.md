**Дата:** 2026-06-27

**Репозиторий:** github.com/Oxana-S/Roni-Site (public, GitHub Pages, subpath `/Roni-Site/`)

## 1. Контекст

Сайт фэнтези-автора Ростислава Вишневского — статический экспорт из no-code-движка **dc-runtime** (`support.js`). Движок на клиенте грузит React/ReactDOM (+ Babel) и рендерит кастомные элементы `<x-dc>`, `<dc-import>`, `<sc-for>` и подстановки `{{ }}`. Каждый `.dc.html` — компонент: шаблон + класс `class Component extends DCLogic` с `renderVals()`.

**Решение пользователя:** код заморожен — новых экспортов от дизайнера не будет. Это делает рефакторинг долговечным.

### Текущие проблемы (по итогам аудита)
- **Кириллица в URL** — все файлы названы кириллицей, навигация ссылается на них (вопрос гигиены URL/GitHub, не работоспособности — движок резолвит и кириллицу через `encodeURIComponent`, support.js:1372).
- **Хардкод стилей** — размеры/цвета/отступы инлайном размазаны по 17 файлам; правки несогласованы и почти невидимы (13→14.5px).
- **Шапка не затемняется на скролле (подтверждено эмпирически на проде)** — меню остаётся на прозрачном фоне и налезает на светлый текст. Точная причина требует диагностики на проде (см. §4.1): `window`-listener в `SiteHeader` (строки 67–69) может не срабатывать, плюс часть страниц (напр. `Главная`) имеет собственную инлайн-логику шапки.
- **Разнобой контейнеров** — `1080`(чаще всего) `/1040/1100/1180px` → блоки визуально не выровнены. Узкие `760/560/460` — это меры читаемого текста, НЕ контейнер.
- **Непропорциональные двухколоночные блоки** — колонка картинки ~250px против широкого текста; прижато по верху.
- **Картинки-заглушки** — реальные файлы есть (HTTP 200), но это плейсхолдеры со служебными подписями имён.
- **Адаптив не выверен** — проверен только десктоп 1483px.

### Подтверждённые факты о движке (сверено с кодом в ходе ревью)
- Компоненты резолвятся: `COMPONENT_DIR + "/" + encodeURIComponent(name) + ".dc.html"`, `COMPONENT_DIR = "."` (support.js:1348, 1372). Путь относителен базовому URL документа.
- React/ReactDOM грузятся с **SRI-integrity** (`REACT_URL`/`REACT_DOM_URL` + `REACT_SRI`/`REACT_DOM_SRI`, support.js:1514–1517, `s.integrity` 1528). **Babel — без SRI** (support.js:1020, 1031–1037).
- `<helmet>` **дедупит** `<link>`/`<meta>`/`<script>` по ключу (тег+href), один `mounted`-Set на документ (support.js:1268–1273) → один theme.css инжектится один раз, **если href байт-в-байт одинаков** во всех компонентах.
- `<dc-import>` поддерживает **слоты** (`props.children`, support.js:584) и **спред-пропсы** (`dcProps` через `Object.assign`, support.js:578). Атрибут `on-lang` (kebab) маппится в проп `onLang`.
- `dcNameFromPath` снимает и `.dc.html`, и `.html` (support.js:82) → страницы можно делать `.html`.
- Движок инжектит собственный **ATOMIC_CSS** слой утилит (support.js:1203) — учитывать при именовании классов.
- При бутстрапе: `fetch(location.href)` (support.js:156) и `fetch(COMPONENT_DIR + name + ".dc.html")` (1373) → **локальная разработка только через HTTP-сервер**, под `file://` fetch блокируется (origin null).
- i18n: язык хранится в **`localStorage('vish_lang')`** (читается в `componentDidMount`, пишется в `setLang`); НЕ в URL. Компоненты язык не хранят — получают пропом `lang` + callback `onLang`.

## 2. Цели и не-цели

### Цели
1. **Единый источник истины для дизайна** — все токены в одном `styles/theme.css`. Правка в одном месте → весь сайт.
2. **Единый источник истины для структуры** — переиспользуемые блоки в `components/` с чёткими контрактами пропсов.
3. **Живой Style Guide** — каталог токенов и блоков реальными компонентами + визуальный регресс-baseline.
4. **Латинские URL** — никакой кириллицы в адресах/именах файлов/путях.
5. **Исправить баги** — шапка-скролл, выравнивание, пропорции колонок.
6. **Адаптив** — 375 / 768 / 1024 / desktop по всем страницам, во всех 4 языках.
7. **Доступность (a11y)** — WCAG 2.1 AA как кросс-секущее требование дизайн-системы (см. §3.8).
8. **Без хардкода** — инлайн-значения → `var(--token)` / классы (с явными исключениями, §7).

### Не-цели (YAGNI)
- Переписывание на чистый HTML/CSS (отклонено — риск сломать i18n/плеер/меню).
- Бэкенд (формы, оплата, автовыдача книги).
- Реальные фото (нужны ассеты от пользователя — отдельная фаза).
- Новый визуальный язык/редизайн — сохраняем эстетику, наводим порядок.
- **Латинизация КОНТЕНТА**: тексты переводов (`CH`/`DICT`), бренд (`Р·ВИШНЕВСКИЙ` и др.) и контент остаются на родных алфавитах. Латинизируются только имена файлов, `href`, пути к ассетам и якоря.
- Полноценная мультиязычная SEO (per-language URL, `hreflang`) — отметить как будущую фазу.

## 3. Архитектура

### 3.1 Подход
**A + локальный React.** Оставляем dc-runtime, сверху — централизованный слой токенов и компонентов. React/ReactDOM/Babel — локально в `assets/vendor/`.

### 3.2 Источник истины для дизайна — `styles/theme.css`
Подключается во все страницы и компоненты. **Двойное подключение:** статический `<link rel="stylesheet" href="styles/theme.css">` в реальном `<head>` каждой страницы (ранняя загрузка, без FOUC) **и** в `<helmet>` компонентов (дедуп по href). Href — каноническая строка `styles/theme.css` (относительный путь, корректен под subpath).

**Иерархия токенов: примитивы → семантика → компонентные.**
- **Примитивы:** палитра-сырьё (`--c-gold-500:#c9a86a`, `--c-ink-900:#14110d`…), шкала `--space-1…12` (4px-базис), радиусы, тени.
- **Семантика:** `--bg`, `--bg-2`, `--gold`, `--gold-soft`, `--text`, `--text-dim`, `--text-mute`, `--line-soft/-med/-strong`, `--header-bg`, `--focus-ring`, `--touch-min:44px`.
- **Типографика:** `--font-display:Forum,serif`, `--font-body:Spectral,serif`; размеры `--fs-eyebrow/-small/-nav/-body/-lead/-h3/-h2/-h1` (крупные — `clamp()`); веса `--fw-300/400/500`; `--lh-*`; трекинг `--tracking-*` (бренд: `letter-spacing:3px/.5px/1px`).
- **Ширины:** `--container` (широкая, целевая 1180px — намеренное расширение ~10 секций, фиксируется как изменение вида на Фазе 3), `--container-narrow`, `--measure` (проза 60–75ch); узкие `760/560/460` маппятся на `--measure`, не на контейнер.
- **Служебные:** `--z-header/-overlay/-toast`, `--opacity-*` / золотая альфа (`rgba(201,168,106,.18/.3/.7)`), `--t-fast/-med` + easing.
- **Тема:** для тёмной-only — плоская семантика поверх примитивов. Если светлый режим реально нужен — через `[data-theme]`/`color-scheme`, семантика маппится на примитивы (заложить структуру, не реализовывать сейчас).

**Базовые стили:** reset, `body`, ссылки, заголовки, **`:focus-visible{ outline: var(--focus-ring); outline-offset:2px }`**, `@media (prefers-reduced-motion: reduce)`.
**Классы-компоненты:** `.container`, `.btn`/`.btn--primary`/`.btn--ghost`, `.card`, `.two-col`, `.eyebrow`, `.section`, `.media-frame`. Не дублировать имена ATOMIC_CSS; строить поверх или сознательно не использовать.

**Методология «инлайн → токен»:** статические инлайны — **извлечь в класс с удалением инлайна** (иначе инлайн выигрывает каскад); динамические (в `renderVals`, напр. `langBtns.style`) — ссылаться на `var()` внутри строки.

### 3.3 Источник истины для структуры — `components/` (с контрактами)
Каркас: `SiteHeader`, `SiteFooter`, `PageTemplate`. Секции: `Hero`, `LexiconCards`, `BookFeature`, `TwoCol`, `AudioPlayer`, `BookGrid`, `Quote` (состав уточняется на Фазе 5 по фактическим повторам).

**Для каждого выносимого блока обязательно:**
- `data-props`-схема (как у `SiteHeader`, support.js parses `data-props`).
- Стратегия i18n: текст приходит **пропсами/слотами** со страницы (предпочтительно для переиспользуемости) либо внутренний `CH` (для глобальных блоков типа header/footer). Решение фиксируется в карточке блока.
- Слоты (`children`) vs скаляры — что параметризуется.

Правка движка: `COMPONENT_DIR = "./components"` (support.js:1348). **Все страницы с `<dc-import>` остаются плоско в корне**, чтобы `./components/` всегда резолвился в `/Roni-Site/components/`.
**Гейт (подтверждённый факт):** во всём репо `<dc-import>` ссылается ТОЛЬКО на `SiteHeader`/`SiteFooter`. Удерживать инвариант: каждый новый блок — только в `components/`. На Фазе 0 — grep-чек «нет dc-import name=, кроме SiteHeader/SiteFooter».

### 3.4 Живой Style Guide — `styleguide.html`
Каталог: палитра **с колонкой contrast-ratio**, типографика, кнопки/карточки во **всех состояниях** (hover/focus/active/disabled), spacing-шкала, все блоки из `components/`, во **всех 4 языках**, do/don't. Таблица токенов **генерируется из theme.css** (не дублирует значения).
**Регрессия:** Playwright-скриншоты styleguide + ключевых страниц как baseline (ручные скриншоты — слабая защита).

### 3.7 Инвариант i18n (фиксируется, чтобы не сломать)
- Ключ языка — **`localStorage('vish_lang')`**, имя НЕ меняется. Чтение в `componentDidMount`, запись в `setLang`.
- При введении `PageTemplate` состояние языка и эти два обращения переносятся в него **целиком** (один источник), не дублируются по страницам.
- Каждый `<dc-import name="SiteHeader">` имеет `lang` и `on-lang`; `SiteFooter` — `lang`. Регресс-тест: клик по языку в шапке меняет язык **и контента, и футера**.
- `document.documentElement.lang` обновляется в `setLang`/`componentDidMount` (для a11y и SEO-минимума).

### 3.8 Доступность (кросс-секущее требование)
- **Контраст WCAG AA:** золото 8.33 ✅, текст 14.9 ✅; **`--text-mute` #8a8070 = 4.84** (впритык) → поднять до ≥ #968b78 или разрешить только для крупного текста; **`--text-dim`** задать явно с таргетом ≥ 4.5.
- **Токены:** `--focus-ring` (`2px solid var(--gold)` + offset), `--touch-min:44px`, `--header-bg` (гарантированная полупрозрачная подложка + blur — читаемость меню и в не-scrolled состоянии).
- **`prefers-reduced-motion`** — глобально гасить анимации/переходы.
- **aria-контракты:** бургер `aria-expanded`+`aria-controls`+`<button>`+`aria-label`, focus-trap и Esc; переключатель языка — `<button>` + `aria-pressed`/`aria-current` + `role="group" aria-label`; `<nav aria-label>` + `aria-current="page"`; **skip-link** к контенту (фикс-шапка перекрывает); AudioPlayer — кнопки `<button>`+`aria-label`, прогресс `role="slider"`/нативный range; формы — `<label for>`, `aria-describedby`/`aria-invalid`, `aria-live` для статуса.
- **alt-тексты** картинок (закрепить в `.media-frame`/`BookGrid`, особенно на Фазе 6).
- Один `<h1>` на страницу, без перепрыгивания уровней.

## 4. Исправления (встроены в рефакторинг)

1. **Шапка-скролл.** Сначала **диагностика на реальном проде** (какой элемент скроллит, чему равен `scrollY`). **Основное решение — IntersectionObserver по невидимому сентинелу** в начале контента (container-agnostic, одинаково на всех страницах); правка `window`-listener — только если диагностика подтвердит, что скроллит `window`. Гарантировать читаемость меню и в верхнем состоянии через `--header-bg`. Свести инлайн-шапки страниц (напр. `Главная`) к компоненту `SiteHeader`.
2. **Единый контейнер.** Секции → `var(--container)` + `margin:0 auto` + `--gutter`; прозовые ширины → `--measure`.
3. **Двухколоночные блоки.** Пропорции ~40/60 (`grid-template-columns`), `align-items:center`, картинка тянется по высоте (`.media-frame`).
4. **Латиница в URL.** Переименование файлов + правка ссылок во ВСЕХ местах: `SiteHeader.NAV[].href`, инлайн-`href` бренда в Header (стр. 33), все инлайн-`href` в шаблоне `SiteFooter`, все инлайн-`href` в шаблонах страниц. **`CH`/`DICT` ссылок НЕ содержат — только тексты.** Сохранять **query-string и якоря** (`Статья.dc.html?id=1 → article.html?id=1`, `#books`, `#read`). Учесть **длинное тире** в ссылке на книгу из `404` (`Книга — Просто…` с `—` vs дефис в имени файла) — проверить grep обоих вариантов.
5. **Адаптив.** 375/768/1024 по каждой странице во всех 4 языках (UK/BG длиннее — проверять переполнение шапки/кнопок). Брейкпоинт 820px в шапке выверить. Брейкпоинты задокументировать как константы ДС.
6. **Заглушки.** Убрать служебные подписи `*.jpg`; реальные фото + alt — отдельной фазой по ассетам пользователя.

## 5. Фазы внедрения (каждая — отдельный коммит и проверка)

- **Фаза 0 — Vendoring & патч движка.**

  - В `support.js`: `REACT_URL/REACT_DOM_URL/BABEL_URL` → относительные локальные пути (`assets/vendor/...`); `COMPONENT_DIR="./components"`.
  - Перенести канонические `SiteHeader`/`SiteFooter` в `components/`; **удалить папку `templates/` (вкл. вторую `support.js`)**, `Шаблон страницы` → `components/PageTemplate.dc.html`.
  - Grep-гейт: нет `<dc-import name=>` кроме `SiteHeader/SiteFooter`.
  - **Проверка:** локальный HTTP-сервер, эмулирующий `/Roni-Site/`; вкладка Network — нет обращений к unpkg, нет 404 на компонентах; визуально как было. Проверить, грузится ли Babel вообще (возможно, для `text/x-dc` не нужен — кандидат на ленивую загрузку, не в этой спеке).
- **Фаза 1 — Токены.** `styles/theme.css` со значениями = текущие (без визуальных изменений) + примитивы/семантика/spacing/типошкала; подключить статически в `<head>` всех страниц и в `<helmet>` компонентов. **Проверка:** вид не изменился, переменные доступны.
- **Фаза 2 — Латинизация.** Сверить `index.html` ↔ `Главная.dc.html`, оставить `index.html` каноном, удалить `Главная.dc.html`, переписать `Главная.dc.html#books → index.html#books`. Переименовать остальные файлы; переписать ВСЕ ссылки (§4.4) с сохранением `?query`/`#anchor`; учесть em-dash. **Проверка:** вся навигация и переключение языков работают (включая футер), кириллицы в URL нет, `?id`-страницы открываются.
- **Фаза 3 — Рефакторинг стилей + фиксы.** Инлайн → токены/классы постранично (удаляя инлайн); фикс шапки (диагностика → IO-сентинел); единый контейнер/меры; пропорции колонок; `--text-mute`/`--text-dim`/`--focus-ring`/`--header-bg`. **Проверка:** визуальная регрессия, шапка затемняется, контраст AA.
- **Фаза 4 — Адаптив + a11y.** 375/768/1024 во всех 4 языках; aria-контракты (§3.8), focus-visible, тач-таргеты, reduced-motion, skip-link, `<html lang>`. **Проверка:** скриншоты брейкпоинтов; клавиатурный проход; контраст.
- **Фаза 5 — Компоненты + Style Guide.** Вынести повторяющиеся секции в `components/` с контрактами пропсов/слотов; перенести state языка в `PageTemplate` (ключ `vish_lang` неизменен); пересобрать `styleguide.html`; завести Playwright-baseline. **Проверка:** страницы используют общие блоки; каталог отражает токены/компоненты/4 языка/состояния.
- **Фаза 6 — Реальные картинки.** Заменить заглушки + alt (нужны ассеты от пользователя).

## 7. Критерии готовности (Definition of Done)
- В URL нет кириллицы; вся навигация, `?id`-страницы и переключение языков (вкл. футер) работают; язык запоминается между страницами.
- Один `theme.css` управляет цветом/типографикой/spacing/контейнером; изменение токена отражается на всём сайте. Хардкод убран, **кроме** декоративных one-off (явно помечены).
- Повторяющиеся блоки живут в `components/` с контрактами пропсов/слотов и переиспользуются.
- `styleguide.html` рендерит токены/блоки реальными компонентами, во всех состояниях и 4 языках; есть Playwright-baseline.
- Шапка читаема всегда и затемняется на скролле; контейнеры выровнены; колонки пропорциональны.
- **a11y:** контраст всех текстовых пар ≥ AA; клавиатурная доступность меню/языков/плеера/форм; `:focus-visible` виден везде; тач-таргеты ≥ 44px; `prefers-reduced-motion`; `<html lang>` обновляется; skip-link; alt у картинок.
- Сайт не обращается к unpkg (React/Babel локальны); `templates/` удалена; единственный `support.js`.
- Корректно на 375/768/1024/desktop.
- Каждая фаза — отдельный коммит в `main`, задеплоено и проверено на проде.
