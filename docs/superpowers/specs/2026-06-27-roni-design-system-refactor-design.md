## 1. Контекст

Сайт фэнтези-автора Ростислава Вишневского — статический экспорт из no-code-движка **dc-runtime** (`support.js`). Движок на клиенте грузит React/ReactDOM + Babel с unpkg, рендерит кастомные элементы `<x-dc>`, `<dc-import>`, `<sc-for>` и шаблонные подстановки `{{ }}`. Каждый `.dc.html` — это компонент: шаблон + класс `class Component extends DCLogic` с `renderVals()`.

**Решение пользователя:** код заморожен — новых экспортов от дизайнера НЕ будет, код теперь наш. Это снимает конфликт «рефакторинг vs ре-экспорт» и делает чистый рефакторинг долговечным.

### Текущие проблемы (по итогам аудита)
- **Кириллица в URL** — все файлы названы кириллицей (`Главная.dc.html`…), все `href` ссылаются на них.
- **Хардкод стилей** — размеры/цвета инлайном размазаны по 17 файлам; правки несогласованы и почти невидимы (13→14.5px).
- **Шапка не затемняется на скролле** — `window.addEventListener('scroll')` (SiteHeader строка 67), но контент скроллится внутри `#dc-root`, `window.scrollY` всегда 0 → класс `.scrolled` не включается → меню налезает на светлый текст.
- **Разнобой контейнеров** — `1040 / 1080 / 1100 / 1180px` на разных секциях → блоки визуально не выровнены.
- **Непропорциональные двухколоночные блоки** — колонка картинки ~250px против широкого текста; прижато по верху, не отцентровано.
- **Картинки-заглушки** — реальные файлы есть (HTTP 200), но это плейсхолдеры со служебными подписями имён поверх.
- **Адаптив не выверен** — проверен только десктоп 1483px.

### Подтверждённые факты о движке
- Компоненты резолвятся: `COMPONENT_DIR + "/" + name + ".dc.html"`, где `COMPONENT_DIR = "."` (support.js:1348, 1372).
- React/Babel грузятся с unpkg: `REACT_URL`, `REACT_DOM_URL` (support.js:1514–1516), `BABEL_URL` (1020).
- Подключение блоков: `<dc-import name="SiteHeader" lang="{{ lang }}" …>`.
- i18n: словари `CH = {ru,en,uk,bg}` внутри каждого компонента; язык — проп, callback `onLang`.
- Стили: микс — общие классы в `<style>` внутри `<helmet>` + инлайн на элементах.

## 2. Цели и не-цели

### Цели
1. **Единый источник истины для дизайна** — все токены (цвет/типографика/размеры/контейнер) в одном `styles/theme.css`. Правка в одном месте → меняется весь сайт.
2. **Единый источник истины для структуры** — переиспользуемые блоки в `components/`; новые страницы собираются из них без страха сломать существующее.
3. **Живой Style Guide** — каталог токенов и блоков, рендерится реальными компонентами.
4. **Латинские URL** — никакой кириллицы в адресах.
5. **Исправить баги** — шапка-скролл, выравнивание, пропорции колонок.
6. **Адаптив** — выверить 375 / 768 / 1024 / desktop по всем страницам.
7. **Без хардкода** — инлайн-значения заменяются на `var(--token)` / общие классы.

### Не-цели (YAGNI)
- Переписывание на чистый HTML/CSS (отклонено — риск сломать i18n/плеер/меню).
- Бэкенд (формы, оплата, автовыдача книги) — отдельная задача, см. README разделы 6/10.
- Реальные фото — нужны ассеты от пользователя (отдельная фаза).
- Новый визуальный язык/редизайн — сохраняем текущую эстетику, наводим порядок.

## 3. Архитектура

### 3.1 Подход
**A + локальный React.** Оставляем dc-runtime (он даёт компоненты + i18n + плеер + меню), сверху строим централизованный слой темы и компонентов. React/ReactDOM/Babel кладём локально в `assets/vendor/`, убирая зависимость от unpkg.

### 3.2 Источник истины для дизайна — `styles/theme.css`
Один файл, подключается во все страницы и компоненты через `<helmet>`:
```html
<link rel="stylesheet" href="styles/theme.css">
```
Содержит:
- `:root{ … }` — токены:
  - **Цвет:** `--bg`, `--bg-2`, `--gold`, `--gold-soft`, `--text`, `--text-dim`, `--text-mute`, `--line-soft/-med/-strong`.
  - **Слой/сетка:** `--container:1180px`, `--gutter:7vw`, `--header-h:66px`, `--radius`, `--shadow`.
  - **Типографика:** `--font-display:Forum,serif`, `--font-body:Spectral,serif`, шкала `--fs-eyebrow`, `--fs-small`, `--fs-nav`, `--fs-body`, `--fs-lead`, `--fs-h3`, `--fs-h2`, `--fs-h1` (крупные — через `clamp()` для адаптива).
  - **Анимации:** `--t-fast`, `--t-med`, easing.
- Базовые стили (reset, `body`, ссылки, заголовки).
- Общие классы-компоненты: `.container`, `.btn`/`.btn--primary`/`.btn--ghost`, `.card`, `.two-col`, `.eyebrow`, `.section`, `.media-frame` и т.д.

Инлайн-хардкод вычищается: статические значения → классы/переменные; динамические инлайны в `renderVals()` ссылаются на переменные (`color:var(--text)`).

### 3.3 Источник истины для структуры — `components/`
Переиспользуемые dc-компоненты:
- Каркас: `SiteHeader`, `SiteFooter`, `PageTemplate`.
- Секции-блоки: `Hero`, `LexiconCards`, `BookFeature`, `TwoCol`, `AudioPlayer`, `BookGrid`, `Quote` (состав уточняется на фазе 5 по факту повторяющихся секций).

Правка движка: `COMPONENT_DIR = "./components"` в `support.js`. Все страницы остаются плоско в корне, чтобы `./components/Name.dc.html` всегда резолвился в `/components/`.

### 3.4 Живой Style Guide — `styleguide.html`
Перестроенная «Дизайн-система»: секции «Палитра» (из `--*` цветов), «Типографика» (шкала `--fs-*`), «Кнопки», «Карточки», «Шаблоны секций» — каждый блок рендерится реальным компонентом из `components/`. Служит и витриной, и регрессионной проверкой.

### 3.5 Структура папок
```

Навигация (`href`) ведёт на `.html`-страницы; `<dc-import name="X">` подтягивает `components/X.dc.html`.

## 4. Исправления (встроены в рефакторинг)

1. **Шапка-скролл.** Слушать скролл реального контейнера: определить элемент прокрутки (`#dc-root` или его потомок) и вешать listener на него; запасной вариант — IntersectionObserver на невидимый сентинел в начале страницы. Цель: `.scrolled` корректно включает тёмную подложку.
2. **Единый контейнер.** Все секции → `var(--container)` + `margin:0 auto` + общий боковой `--gutter`. Убрать `1040/1080/1100`.
3. **Двухколоночные блоки.** Пропорции ~40/60 (`grid-template-columns`), `align-items:center`, картинка тянется по высоте текста (`.media-frame`).
4. **Латиница в URL.** Переименование файлов + правка всех `href` в `SiteHeader.NAV`, `SiteFooter`, внутри страниц и `CH`-словарях. Внутренние якоря (`#books`, `#read`) сохраняются.
5. **Адаптив.** Проход по каждой странице на 375/768/1024: контейнер, `clamp()`-типографика, стек колонок на мобиле, масштаб картинок, тач-таргеты. Брейкпоинт шапки 820px уже есть — выверить.
6. **Заглушки.** Убрать служебные подписи `*.jpg`; реальные фото — отдельной фазой по ассетам пользователя.

## 5. Фазы внедрения (каждая — отдельный коммит и проверка)

- **Фаза 0 — Vendoring & патч движка.** Скачать React/ReactDOM/Babel в `assets/vendor/`; в `support.js` переключить `REACT_URL/REACT_DOM_URL/BABEL_URL` на локальные и `COMPONENT_DIR="./components"`. Перенести `SiteHeader`/`SiteFooter` в `components/`. **Проверка:** сайт грузится и работает без обращений к unpkg (вкладка Network).
- **Фаза 1 — Токены.** Создать `styles/theme.css` со значениями = текущие (визуально без изменений); подключить `<link>` во все страницы/компоненты. **Проверка:** внешний вид не изменился, переменные доступны.
- **Фаза 2 — Латинизация.** Переименовать файлы, переписать все ссылки/`dc-import`/`CH`. **Проверка:** вся навигация по страницам и языкам работает, кириллицы в URL нет.
- **Фаза 3 — Рефакторинг стилей.** Инлайн → токены/классы постранично; фикс шапки-скролла; единый контейнер; пропорции колонок. **Проверка:** визуальная регрессия по страницам, шапка затемняется.
- **Фаза 4 — Адаптив.** 375/768/1024 по всем страницам. **Проверка:** скриншоты брейкпоинтов.
- **Фаза 5 — Компоненты + Style Guide.** Вынести повторяющиеся секции в `components/`; пересобрать `styleguide.html` как живой каталог. **Проверка:** страницы используют общие блоки; каталог отражает реальные токены/компоненты.
- **Фаза 6 — Реальные картинки.** Заменить заглушки (нужны ассеты от пользователя).

## 6. Риски и смягчение

| Риск | Смягчение |
|---|---|
| Патч `support.js` ломает резолвинг компонентов | Все dc-import-страницы держим плоско в корне; проверка после фазы 0 до остального. |
| Латинизация рвёт внутренние ссылки/якоря | Полная карта переименований (§3.6); поиск всех `href`/`dc-import` по репо; прогон навигации. |
| Локальный Babel замедляет загрузку (транспиляция в браузере) | Приемлемо на текущем этапе; пред-транспиляция — потенциальная будущая оптимизация (не в этой спеке). |
| Рефакторинг инлайнов меняет вид незаметно | Фаза 1 фиксирует значения = текущие; визуальная регрессия скриншотами после каждой фазы. |
| GitHub Pages subpath `/Roni-Site/` ломает абсолютные пути | Используем относительные пути; проверяем на проде, не только локально. |

## 7. Критерии готовности (Definition of Done)
- В URL нет кириллицы; вся навигация и переключение языков работают.
- Один `theme.css` управляет цветами/типографикой/контейнером; изменение токена отражается на всём сайте.
- Повторяющиеся блоки живут в `components/` и переиспользуются страницами.
- `styleguide.html` рендерит токены и блоки реальными компонентами.
- Шапка корректно затемняется на скролле; контейнеры выровнены; колонки пропорциональны.
- Страницы корректны на 375/768/1024/desktop.
- Сайт не обращается к unpkg (React/Babel локальны).
- Каждая фаза — отдельный коммит в `main`, задеплоено и проверено на проде.
